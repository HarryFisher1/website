About one year into my PhD I decided that I was going to use R Markdown to write my thesis. At the time this was quiet a big decision as I had only recently started using R and playing with R Markdown, however I saw the potential and knew it was a risk worth taking to learn.

I am currently preparing the final draft of my PhD for submission it though it would be good to reflect on what I’ve learnt, and share some tips and tricks I’ve picked up on the way that someone else might find useful.

When I first started, Yihui Xie bookdown book was invaluable and got me pointing in the right direction. I also found a blog post by Ed Berry that had lots of useful tricks for writing a thesis specifically.

I did the bulk of my writing and editing with these two resources and it wasn’t until I had about 6 months left that I discovered papaja by Frederik Aust. papaja (short for ‘preparing APA journal articles’) builds on elements from bookdown to produce documents formatted according to American Psychological Association (APA) style. Before finding papaja, I was trying to do much of the formatting manually, and while this was a good learning experiencing, having a package that took care of most of the formatting was fantastic.

So this post is mainly going to be covering some tricks and tips I have learnt using bookdown and papaja to produce nicely formatted documents in line with the APA manual that both you (and your supervisors) will be happy with! I’m assuming you have some familiarity with how R and R Markdown work and it not I would highly recommend look at a quick tour of what it can do.

YAML front matter

After installing the necessary packages and getting things set up, you can select a papaja template when creating a new .Rmd file. You’ll be greated with several YAML options:

title             : 
shorttitle        : 

author: 
  - name          : "blank"
    affiliation   : ""
    corresponding : yes    # Define only one corresponding author
    address       : "-"
    email         : "-"

affiliation:
  - id            : ""
    institution   : ""

authornote: 
abstract: 
  
keywords          : ~
wordcount         : ~

bibliography      : [library.bib]

floatsintext      : yes
figurelist        : yes
tablelist         : yes
footnotelist      : no
linenumbers       : no
mask              : no
draft             : no

documentclass     : "apa6"
classoption       : "man"
output: 
    papaja::apa6_pdf: default
    #papaja::apa6_docx: default

I found that I really didn’t need to use many of these options, and they are likely more useful for when writing a journal paper in APA style. You also only need to do this once for the main document if you are following the recommended papaja workflow (e.g., one .Rmd file for each chapter in the thesis). I did however make use of a few options…

For my references, I used Mendelay as a organiser and exported the reference information as a .bib file which was saved in my thesis project folder as library.bib.

bibliography      : [library.bib]

As of the latest APA manual, figures and tables are supposed to go at the end of the document, however this isn’t really necessary for a thesis and thankfully papaja has an option keep tables in text using:

floatsintext      : yes

Sometimes figures and tables can move from where you want them to be, but I found setting fig.pos = 'H' in the chunk options to be pretty good at getting them to stay where you want.

For the output, my final document was in PDF format where papaja really shines, however it does also do fairly good job at getting things right in word too. This was particularly useful for my supervisors who used word to make changes and comments on the different chapter drafts. Swapping between the outputs just required commenting/uncommenting one of these lines:

output: 
    papaja::apa6_pdf: default
    #papaja::apa6_docx: default

At the end of my YAML I also had this section:

header-includes:
   - \raggedbottom
   - \usepackage{setspace}
   - \AtBeginEnvironment{tabular}{\onehalfspacing}
   - \AtBeginEnvironment{lltable}{\onehalfspacing}
   - \captionsetup[table]{font={stretch=1}}
   - \captionsetup[figure]{font={stretch=1}}
   - \setlength{\skip\footins}{1cm}
   - \setlength{\footnotesep}{0.5cm}

This included some additional control over the output of the document which I borrowed from various examples I came across. For example, -\raggedbottom prevents LaTeX from varying the spacing between paragraphs to minimise empty space on pages, -\AtBeginEnvironment{tabular}{\onehalfspacing} uses 1.5 line spacing in tables vs the standard 2.0 in text, -\captionsetup[table]{font={stretch=1}} sets the table title to 1.0 line spacing etc..

This article by Tobias Heycke provides some really useful information for additional latex packages that you can also include in this section.

Title page

Contents pages

The able of contents, list of tables and list of figures can be easily included by simply using the

\tableofcontents

\listoftables

\listoffigures

commands in your main document. You can separate them by using the \clearpage command.

The rest of the document

If you’re familiar with using R Markdown you can write the chapters like you would any other document. Once they are down you can include the following line in a code chunk in your main document:

{child="01-intro.Rmd"}

Now when you knit the main document, the introduction chapter (called “01-intro.Rmd” in this case) will be included after your title pages. Papaja will also automatically generates tables of contents depending on your the headers in your other files, as well as number all tables and figures.
